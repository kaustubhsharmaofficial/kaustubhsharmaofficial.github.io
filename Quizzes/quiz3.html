AWS Developer Quiz-3

  1. A developer is creating a new application that will store data in a DynamoDB table. Which APIs can be used to read, write and modify individual items in the table?





    2. WHY A IS INCORRECT? CONCEPT: BatchGetItem & BatchWriteItem BatchGetItem and BatchWriteItem are used when you have multiple items to read/write. In this scenario we are updating individual items.
    WHY B IS INCORRECT? CONCEPT: DeleteItem DeleteItem will delete single items in a table by primary key. We do not want to delete, we want to modify so UpdateItem should be used instead.
    WHY C IS INCORRECT? CONCEPT: TransactWriteItems TransactWriteItems is a synchronous write operation that groups up to 25 action requests. In this scenario we are updating individual items.or Tasks.
    WHY D IS CORRECT? CONCEPT: GetItem, PutItem, UpdateItem

    GetItem operation returns a set of attributes for the item with the given primary key. If there is no matching item, GetItem does not return any data and there will be no Item element in the response.

    PutItem creates a new item or replaces an old item with a new item. If an item that has the same primary key as the new item already exists in the specified table, the new item completely replaces the existing item.

    UpdateItem edits an existing item’s attributes or adds a new item to the table if it does not already exist. You can put, delete, or add attribute values. You can also perform a conditional update on an existing item

    Short Trick There is two Keywords in Question:

    1."Individual items" which clearly denied Batch & Transact Commands, as these are used for multiple items.

    2."read, write and modify" which clearly denied Delete Command, thus remaining with only option D.
    References: AWS DOCS

  2. A developer needs to implement a caching layer in front of an Amazon RDS database. If the caching layer fails, it is time consuming to repopulate cached data so the solution should be designed for maximum uptime. Which solution is best for this scenario?





    3. WHY A IS INCORRECT? CONCEPT:RedShift RedShift is a data warehouse for use in online analytics processing (OLAP) use cases. It is not suitable to be used as a caching layer.
    WHY B IS INCORRECT? CONCEPT: Memcached Memcached does not offer read replicas or auto-failover and therefore cannot provide high availability.
    WHY C IS INCORRECT? CONCEPT: DAX DAX is used in front of DynamoDB, not Amazon RDS.
    WHY D IS CORRECT? CONCEPT: Implement Amazon ElastiCache Redis Amazon ElastiCache provides fully managed implementations of two popular in-memory data stores – Redis and Memcached. ElastiCache is a web service that makes it easy to deploy and run Memcached or Redis protocol-compliant server nodes in the cloud. The in-memory caching provided by ElastiCache can be used to significantly improve latency and throughput for many read-heavy application workloads or compute-intensive workloads. It is common to use ElastiCache as a cache in front of databases such as Amazon RDS. The two implementations, Memcached, and Redis, each offer different capabilities and limitations.Only Redis supports read replicas and auto-failover. The Redis implementation must be used if high availability is required, as is necessary for this scenario. Therefore the correct answer is to use Amazon ElastiCache Redis.
    Short Trick There are two Keywords in Question:

    1."caching layer in front of an Amazon RDS" indicates towards ElastiCache, i.e. Option B & D,

    2."maximum uptime" clearly signifies towards to Amazon ElastiCache Redis.
    References: AWS DOCS

  3. A developer is planning to launch as serverless application composed of AWS Lambda, Amazon API Gateway, and Amazon DynamoDB. What is the EASIEST way to deploy the application using simple syntax?





    4. WHY A IS INCORRECT? CONCEPT: Serverless Application Repository Serverless Application Repository is a managed repository for serverless applications.
    WHY B IS CORRECT? CONCEPT: AWS Serverless Application Model (SAM) AWS Serverless Application Model (SAM) is an open-source framework for building serverless applications. It provides shorthand syntax to express functions, APIs, databases, and event source mappings. With just a few lines per resource, you can define the application you want and model it using YAML. During deployment, SAM transforms and expands the SAM syntax into AWS CloudFormation syntax, enabling you to build serverless applications faster. To get started with building SAM-based applications, use the AWS SAM CLI. SAM CLI provides a Lambda-like execution environment that lets you locally build, test, and debug applications defined by SAM templates. You can also use the SAM CLI to deploy your applications to AWS.

    WHY D IS INCORRECT? CONCEPT: AWS Elastic Beanstalk Elastic Beanstalk cannot be used to deploy Lambda, API Gateway or DynamoDB.
    Short Trick There is one Keyword in Question:

    1."Read the EASIEST way to deploy the application using simple syntax" clearly indicates AWS Serverless Application Model (SAM) as it is made for this purpose only & easy one.
    References: AWS DOCS

  5. A company is running an order processing system on AWS. Amazon SQS is used to queue orders and an AWS Lambda function processes them. The company recently started noticing a lot of orders are failing to process. How can a Developer MOST effectively manage these failures to debug the failed orders later and reprocess them, as necessary?





    6. WHY A IS INCORRECT? CONCEPT: Amazon SNS topic There is no way to isolate messages that have failed to process when subscribing an SQS queue to an SNS topic.
    WHY B IS INCORRECT? CONCEPT: Amazon CloudWatch Logs SQS does not publish message success/failure to CloudWatch Logs.
    WHY C IS INCORRECT? CONCEPT: AWS CloudTrail logs CloudTrail records API activity not performance metrics or logs.
    WHY D IS CORRECT? CONCEPT: dead-letter queues Amazon SQS supports dead-letter queues, which other queues (source queues) can target for messages that can't be processed (consumed) successfully. Dead-letter queues are useful for debugging your application or messaging system because they let you isolate problematic messages to determine why their processing doesn't succeed. The Developer should therefore implement dead-letter queues for failed orders from the order queue. This will allow full debugging as the entire message is available for analysis.
    Short Trick There are two Keywords in Question:

    1."Collect system-level metrics from on-premises servers & EC2 instances" &

    2."Most efficiently"

    redirects to CloudWatch agent can be installed on both on-premises servers and Amazon EC2 instances & it is most efficient also.
