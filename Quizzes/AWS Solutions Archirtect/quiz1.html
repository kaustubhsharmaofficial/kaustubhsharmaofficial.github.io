  3. A Developer is creating multiple AWS Lambda functions that will be using an external library that is not included in the standard Lambda libraries. What is the BEST way to make these libraries available to the functions?





    4. WHY A IS INCORRECT? CONCEPT: External library within function code You should not include an external library within the function code. Even if possible this would result in bloated code that could slow down execution time.
    WHY B IS INCORRECT? CONCEPT: S3 & function code This would likely result in increased latency of our function execution. Instead we should either package the library in the deployment package for our function or use layers.
    WHY C IS CORRECT? CONCEPT: Lambda include external library/Layers You can configure your Lambda function to pull in additional code and content in the form of layers. A layer is a ZIP archive that contains libraries, a custom runtime, or other dependencies. With layers, you can use libraries in your function without needing to include them in your deployment package. Layers let you keep your deployment package small, which makes development easier. One of the best practices for AWS Lambda functions is to minimize your deployment package size to its runtime necessities in order to reduce the amount of time that it takes for your deployment package to be downloaded and unpacked ahead of invocation. Therefore, it is preferable to use layers to store the external libraries to optimize performance of the function.
    WHY D IS INCORRECT? CONCEPT: Deployment package include external library Best practice is to minimize package sizes to runtime necessities. Also, this would require including the library in all function deployment packages which is more time consuming & increase package size whereas with layers we can create a single layer that is used by all functions.
    Short Trick There is one Keyword in Question:

    1."using an external library" & to include external library, best practice is to use configuration-layers in Lambda.
    References: AWS DOCS

  4. A company uses an Amazon Simple Queue Service (SQS) Standard queue for an application. An issue has been identified where applications are picking up messages from the queue that are still being processed causing duplication. What can a Developer do to resolve this issue?





    5. WHY A IS INCORRECT? CONCEPT: ReceiveMessageWaitTimeSeconds ReceiveMessageWaitTimeSeconds is the length of time, in seconds, for which a ReceiveMessage action waits for a message to arrive. This is used to configure long polling.
    WHY B IS CORRECT? CONCEPT: VisibilityTimeout When a consumer receives and processes a message from a queue, the message remains in the queue. Amazon SQS doesn't automatically delete the message. Because Amazon SQS is a distributed system, there's no guarantee that the consumer actually receives the message (for example, due to a connectivity issue, or due to an error in the consumer application). Thus, the consumer must delete the message from the queue after receiving and processing it.

    Immediately after a message is received, it remains in the queue. To prevent other consumers from processing the message again, Amazon SQS sets a visibility timeout, a period of time during which Amazon SQS prevents other consumers from receiving and processing the message. The default visibility timeout for a message is 30 seconds. The minimum is 0 seconds. The maximum is 12 hours.
    Therefore, the best thing the Developer can do in this situation is to increase the VisibilityTimeout API action on the queue
    WHY C IS INCORRECT? CONCEPT: DelaySeconds DelaySeconds controls the length of time, in seconds, for which the delivery of all messages in the queue is delayed.
    WHY D IS INCORRECT? CONCEPT: RedrivePolicy RedrivePolicy is a string that includes the parameters for the dead-letter queue functionality of the source queue as a JSON object.
    Short Trick There is one Keyword in Question:

    1."duplication" & SQS message duplication can be removed by increasing Visibility Timeout.
    References: AWS DOCS

  5. An Auto Scaling Group (ASG) of Amazon EC2 instances is being created for processing messages from an Amazon SQS queue. To ensure the EC2 instances are cost-effective, a Developer would like to configure the ASG to maintain aggregate CPU utilization at 70%. Which type of scaling policy should the Developer choose?





    6. WHY A IS INCORRECT? CONCEPT: Scheduled Scaling Policy This is used to schedule a scaling action at a specific time and date rather than dynamically adjusting according to load.
    WHY B IS CORRECT? CONCEPT: Target Tracking Scaling Policy With target tracking scaling policies, you select a scaling metric and set a target value. Amazon EC2 Auto Scaling creates and manages the CloudWatch alarms that trigger the scaling policy and calculates the scaling adjustment based on the metric and the target value. The scaling policy adds or removes capacity as required to keep the metric at, or close to, the specified target value. In addition to keeping the metric close to the target value, a target tracking scaling policy also adjusts to the changes in the metric due to a changing load pattern.
    For example, you can use target tracking scaling to:
    1. Configure a target tracking scaling policy to keep the average aggregate CPU utilization of your Auto Scaling group at 70 percent.
    2. Configure a target tracking scaling policy to keep the request count per target of your Elastic Load Balancing target group at 800 for your Auto Scaling group.
    WHY C IS INCORRECT? CONCEPT: Simple scaling policies In Simple Autoscaling Policy, you define the threshold value and upper bound. The alarm monitors the metric values. When the metric value breaches the threshold, the scaling policy acts. The instances are added or removed as per the policy.
    WHY D IS INCORRECT? CONCEPT: Step scaling policies Step scaling policies increase or decrease the current capacity of a scalable target based on a set of scaling adjustments, known as step adjustments. The adjustments vary based on the size of the alarm breach. All alarms that are breached are evaluated by Application Auto Scaling as it receives the alarm messages.
    Short Trick There are two Keywords in Question:

    1."maintain aggregate CPU utilization at 70%" redirects to Target tracking scaling policy as we have to maintain capacity at specified target.
